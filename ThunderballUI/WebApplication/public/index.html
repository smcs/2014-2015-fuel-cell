Project Goals

Posted on April 18th 2015 by Matthew Flathers

The goal this spring was to make a user-friendly web interface that communicates directly to a fuel cell produced by Protonex Technology Inc. On this site I hoped to accumulate and analyze data in a way that would be easy and accessible to clients, without taking up too much storage on the fuel cell with old and uneccessary data.

The Process

Posted on May 6th 2015 by Matthew Flathers

Once I figured out the basics of html programming, editing a template bacground to create the graphics of this site was not super challenging. However, the trial and error process of learning how html worked was more time consuming and challenging than I expected it to be. Creating the home page and the stress relief pages were fairly straightforward, with very little complicated code involved in adding text or pictures. The event log page was a little more complicated. I had to dig through the code I had written to back up my first attempt at a web site and figure out how to fit the auto-generating chart into the template that I was working with. Once I successfully added and formatted that, I had to figure out how to create buttons on the page and write the code through multiple layers of the code to get from the client side to the server side in order to make them funtional. However, while challenging, that was nothing compared to the difficulty of the statistics page. Figuring out how to take the data out of a 250,000 line csv file was challenging in itself, and then I spent a LONG time trying to parse and manipulat the data in a way so that my database would not have 250,000 entries. As soon as I got to the data piece, however, it was clear that making a live updating chart with a stream of data directly from the fuel cell might be more than I could handle. I suceeded in creating hourly displays for Energy and Fuel Consumption, but all attempts to make it update live failed miserably.

Success?

Posted on May 19th 2015 by Matthew Flathers

I was largely successful on the web server element of the project, and while I may not have completed all I hoped to on the data section, i learned far more about data manipulation, c++, and connecting multiple layers of code than I expected to when i set out.
